# 影响范围分析功能增强执行计划

本文档为 CodeSense AI 插件中关于「影响范围分析报告（向上调用链总结、受影响入口点汇总、LLM 模型名称展示）」功能的优化执行计划。

## 需求背景与目标

1. **链路编号一致性**: 在报告的“向上调用链总结表”中添加链路编号，使得表格内容能与上方树状输出的链路（例如 `链路 1:`）完美对应。
2. **完整覆盖顶层入口**: 确保向上调用链的所有“最上层”（没有上一级调用者的节点）都必须展示在「受影响入口点汇总」中。如果没有匹配到具体的注解（HTTP接口/RPC/定时任务/MQ等），统一归类到特殊的 **其他**（`OTHER`）类别中。
3. **元信息模型展示**: 在“分析元信息”中，从代码插件配置提取并正确展示当前使用的 LLM 模型名称。
4. **AI 说明生成方案**: 讨论将「受影响入口点」一栏内的 `*待 AI 生成*` 占位符替换为真实的 AI 总结的可能方案。

## 拟定变更详细说明

### 1. `models.kt`

- **[MODIFY]** `EntryPointType` 枚举：
  增加 `OTHER` 类型作为 fallback（例如：`OTHER("⚙️", "其他顶级方法")`），以涵盖那些没有特定框架注解的链路顶层。

### 2. `CallTree` 实体类内的入口点收集逻辑 (`models.kt`)

- **[MODIFY]** `collectEntryPoints` 方法：
  当遍历时，如果当前节点 `children.isEmpty()`（代表它已经是当前分析深度的最上层）并且 `!isEntryPoint` 时，主动将它转换为 `EntryPointType.OTHER`，收集进入影响的入口点列表。这样就能兑现“所有最上层都要展示”的承诺。

### 3. `ReportGenerator.kt`

- **[MODIFY]** `generateSingleMethodReport`:
  重构「向上调用链总结表」逻辑：不再单纯使用 `flattenTree()`，而是迭代 `biTree.callerTree.collectChainPaths()`。以此给出一个携带**【链路编号】**的展现：
  
  | 链路编号 | 链路深度 | 最上层调用者 | 入口类型 | 调用方式 |
  |--------|--------|------------|---------|---------|
  | 1      | 3      | `XxxController` | 🌐 HTTP API | 直接调用 |

- **[MODIFY]** `renderCallerTree`: 
  略微对齐输出格式，确保表格和此处的树完全吻合。

### 4. `AnalyzeImpactAction.kt` (元数据补充)

- **[MODIFY]** 组装 `ImpactReport`：
  从配置里面获取并写入当前的 llm 模型信息作为 metadata：
  ```kotlin
  val settings = CodeSenseSettings.getInstance()
  val provider = settings.getDefaultProvider()
  val report = ImpactReport(
      // ...
      metadata = mapOf("llmModel" to provider.displayName)
  )
  ```

## 用户讨论项：受影响入口点【AI 说明】自动填充方案

> ⚠️ 目前报告中“受影响入口点汇总”表的“AI 说明”列显示为 `*待 AI 生成*`。考虑到流式返回和性能，此处有以下两种方案，**请您回复确认倾向于哪种**：

**方案 A（一次性 Prompt 方案 - 推荐，性能好）**
- 发请求时扩充现有的整体风险评估 Prompt：要求 LLM **首先**针对每一个“受影响入口点”生成一句话的影响说明，**最后**输出总体的风险评估。
- 系统收到流式响应后进行简单的字符串处理（解析出入口说明），分别利用正则表达式替换 Markdown 文本中对应的 `*待 AI 生成*`，同时将总评估渲染到下方。

**方案 B（双重并发请求方案 - 更精准但耗时略长）**
- 异步发出**两个（或多个）请求**。
- 一个请求由 LLM 专职生成“入口点对应的说明”（可传精简上下文）。完成后批量替换表中占位符。
- 另一个请求专注做原有底部的整体代码变更分析。两者并行。

## 验证计划

1. 确认修改 HTTP / 服务类或普通类。
2. 运行 `./gradlew runIde`，触发 Analyze Impact 分析，验证：
   - [ ] 【元信息】正确显示 `MiniMax` 或 `GLM` 模型名称。
   - [ ] 【调用链表格】列出现“链路编号”，并与代码上方的控制台字符串一一对应。
   - [ ] 【受影响入口】未匹配到注解的最上端（如某些工具类或基础 Action）展示 `⚙️ 其他顶级方法` 并列在表中。
   - [ ] 按照讨论敲定的 AI 生成方案，替换相应的表格占位符。
